# chinese-wc2026-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以资源文件形式存放，不针对任何特定域名或具体网站。

## 项目简介

这是一个以 HTML 页面为主要内容的资源仓库。页面以静态形式保存，便于访问、分发和长期归档。仓库本身不提供搜索、导航或动态服务，仅作为页面文件的存储和索引入口。

## 目录结构

```
.
├── README.md               # 本文件
├── pages/                  # 页面文件存放目录
│   ├── page-example.html   # 示例页面（可替换）
│   └── ...                 # 其他独立 HTML 页面
└── assets/                 # 资源文件目录（如 CSS、JS、图片等）
    └── ...
```

- `pages/`：存放所有独立的 HTML 页面文件。每个页面应当自包含，或通过相对路径引用 `assets/` 中的资源。
- `assets/`：存放页面共用的或独立的样式、脚本、图片等资源文件。

## 页面归档说明

- 每个 HTML 页面均为独立文件，可直接在浏览器中打开使用。
- 页面内容不涉及任何具体域名、网站或服务，仅为通用资源或示例用途。
- 页面命名应简洁明了，便于识别和引用。
- 推荐在页面内使用注释或元数据标明归档日期、版本或简要说明。

## 维护说明

- 仓库维护者应定期检查页面文件的可用性和兼容性。
- 如需新增页面，请将 HTML 文件放入 `pages/` 目录，并确保引用的资源路径正确。
- 不建议在页面中嵌入外部依赖或跨域资源，以免影响离线访问或长期可用性。
- 如发现页面损坏或链接失效，欢迎提交 Issue 或 Pull Request。

## 贡献

欢迎提交页面文件或改进建议。请确保：
- 文件格式为标准的 HTML5。
- 不包含任何形式的推广、广告或恶意内容。
- 遵循本仓库的目录结构和命名规范。
